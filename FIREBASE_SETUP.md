# Firebase Setup Guide — VibzCheck

> **MANDATORY**: Complete every step below before running the app.
> The app will crash at launch if Firebase is not configured.

---

## 1. Create Firebase Project

1. Go to [Firebase Console](https://console.firebase.google.com/)
2. Click **Add project** → name it `VibzCheck`
3. Enable or skip Google Analytics (optional)
4. Click **Create project**

---

## 2. Register Flutter Apps

### Android
1. In Firebase Console → Project Overview → click **Android** icon
2. Package name: `com.vibzcheck.app`
3. App nickname: `VibzCheck Android`
4. Click **Register app**
5. Download `google-services.json`
6. Place it at: `android/app/google-services.json`

### iOS (optional)
1. Click **iOS** icon → Bundle ID: `com.vibzcheck.app`
2. Download `GoogleService-Info.plist`
3. Place it at: `ios/Runner/GoogleService-Info.plist`

---

## 3. Configure Android Build Files

### android/build.gradle.kts — add to `plugins` block:
```kotlin
id("com.google.gms.google-services") version "4.4.2" apply false
```

### android/app/build.gradle.kts — add to `plugins` block:
```kotlin
id("com.google.gms.google-services")
```
And set:
```kotlin
defaultConfig {
    applicationId = "com.vibzcheck.app"
    minSdk = 23   // Required for Firebase Auth
    targetSdk = 34
}
```

---

## 4. Install FlutterFire CLI and Initialize

```bash
dart pub global activate flutterfire_cli
flutterfire configure --project=<your-firebase-project-id>
```

This generates `lib/config/firebase_options.dart`.

## 5. Enable Authentication

1. Firebase Console → **Authentication** → **Get started**
2. Sign-in method → **Email/Password** → Enable → Save
3. (Optional) Enable **Google Sign-In**

---

## 6. Create Firestore Database

1. Firebase Console → **Firestore Database** → **Create database**
2. Start in **production mode**
3. Region: `us-central1`
4. Deploy security rules from `firestore.rules`:
   ```bash
   firebase deploy --only firestore:rules
   ```

### Initial Collections to Create Manually (optional for demo)
- `users`
- `sessions`
- `recommendations`

Sub-collections are created automatically by the app:
- `sessions/{id}/songs`
- `sessions/{id}/votes`
- `sessions/{id}/messages`

---

## 7. Firebase Storage (profile photos)

1. Firebase Console → **Storage** → **Get started**
2. Start in production mode, same region as Firestore
3. Deploy storage rules:
   ```bash
   firebase deploy --only storage
   ```

---

## 8. Firebase Cloud Functions + Spotify Setup

### Install Firebase CLI
```bash
npm install -g firebase-tools
firebase login
```

### Initialize Functions
```bash
firebase init functions
# Select: JavaScript, ESLint: Yes, install dependencies: Yes
```

### Copy functions code
The `functions/` folder in this repo contains the Spotify proxy.

### Configure Spotify Credentials
1. Go to [Spotify Developer Dashboard](https://developer.spotify.com/dashboard)
2. Create an app → get `CLIENT_ID` and `CLIENT_SECRET`
3. Store them securely:
   ```bash
   # TODO: Replace with your actual credentials
   firebase functions:config:set spotify.client_id="YOUR_CLIENT_ID" spotify.client_secret="YOUR_CLIENT_SECRET"
   ```

### Deploy Functions
```bash
firebase deploy --only functions
```

## 9. Firebase Cloud Messaging (optional)

1. Firebase Console → **Cloud Messaging** → **Get started**
2. For Android: the `google-services.json` already includes the config
3. For iOS: upload APNs certificate or key

---

## 10. Firebase Emulator (local testing)

```bash
firebase init emulators
# Select: Auth, Firestore, Functions, Storage
firebase emulators:start
```

In `lib/config/firebase_options.dart`, uncomment the emulator lines.
